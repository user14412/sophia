### 常用的 Commit 前缀速查表

除了上面这三个最核心的，还有几个你日常开发经常会用到的：

- **`feat`**: 新增功能 (Feature)
- **`fix`**: 修复 Bug
- **`docs`**: 只改动了文档（比如更新了 README.md 或 API 文档）
- **`style`**: 代码格式变动，不影响逻辑（比如去掉了多余的空格、格式化了 JSON、规范了变量命名缩进）
- **`refactor`**: 重构（既不新增功能，也不修改 bug 的代码变动）
- **`perf`**: 性能优化 (Performance，比如你把之前同步下载图片改成了 `asyncio` 并发下载)
- **`test`**: 增加或修改测试用例
- **`chore`**: 构建过程或辅助工具的变动（比如更新了 `requirements.txt` 里的依赖版本，或者加了 `.gitignore` 文件。你提到的 `ignore` 和 `dev` 通常可以归类到这里）。

### 进阶小技巧：带作用域的提交 (Scope)

当你以后代码越来越多，可以在前缀后面加一个小括号，标明这次改动影响的是哪个模块，这样看起来会极其专业：

- `feat(memory): 增加基于时间戳的记忆衰减机制`
- `fix(voice_node): 解决音色参数为空时的异常崩溃`
- `refactor(plan_node): 优化大模型 prompt 的结构化输出`
