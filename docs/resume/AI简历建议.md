# 项目保研档案：自动化教育视频生成系统

## 一、项目概述

### 项目名称
**基于 LangGraph + RAG 的自动化教育视频生成系统**（v3.0 版本）

### 项目简介
本项目设计并实现了一套从**内容生成→音视频处理→视频合成** 的完整自动化流水线，用于自动生成教育类对谈短视频。系统采用分层架构，支持：
- **播客特化管线**（v3.0）：从参考章节自动拆解粗课题→编导对谈方案→生成双人播客脚本
- **通用管线**（v2.1 兼容）：灵活支持其他视频内容生成场景

### 项目亮点（简历要点）
1. **完整的工程化管线**：从文本→结构化内容→音视频处理→最终视频，展示全栈工程能力
2. **多轮迭代版本演进**（v1.0→v2.1→v3.0）：体现需求分析和系统重构的能力
3. **复杂 Prompt 工程**：涉及多层次、多角色的 Prompt 设计与优化
4. **分布式内容生成**：使用 LangGraph 框架实现复杂流程编排和动态路由
5. **RAG 系统集成**：知识库构建、检索增强生成、多种查询策略组合

## 二、技术选型与架构设计

### 2.1 核心技术栈

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **流程编排** | LangGraph | 原生支持状态机、条件路由、checkpoint persistence；相比 Airflow/Prefect，更适合复杂 AI 工作流 |
| **LLM 接口** | LangChain | 统一的 LLM 调用协议；良好的 structured output 支持 |
| **大模型** | Deepseek Chat | 成本低廉、多语言支持、性能稳定 |
| **RAG 向量库** | Chroma | 轻量级本地部署；支持元数据、持久化存储；适合研究场景 |
| **Embeddings** | HuggingFace m3e-base | 中英双语支持、轻量级、开源 |
| **音频生成** | GPT-SoVITS | 开源、本地部署、多人物角色定制 |
| **视频处理** | MoviePy | 纯 Python、易于集成、支持复杂合成操作 |
| **异步框架** | AsyncIO | 原生 Python 支持、与 LangGraph 无缝集成 |

### 2.3 关键设计决策

#### (1) 动态路由 vs 静态管线
**决策**：采用 LangGraph 的 `Command(goto=...)` 动态路由

**理由**：
- 通用管线和播客特化管线能够满足不同用户需求
- 避免代码重复，通过配置标志位 `enable_podcast_specialization` 切换
- 错误恢复和人类在环审批时，能灵活回到之前的节点

#### (2) RAG 独立为服务层
**决策**：RAG 组件玩全局懒加载，不进入 LangGraph checkpoint

**理由**：
- checkpoint 的序列化/反序列化问题：Chroma/Embeddings 对象无法被 pickle 序列化
- 降低磁盘占用：RAG 组件复杂而体积大
- 实际上应该独立为微服务

#### (3) 三阶段内容生成（Topic → Director → Agent Speechers）
**决策**：将播客脚本生成分解为三个独立阶段

**理由**：
- **Topic**：语义分解，AI 拆解章节为粗课题（零基础→深度的递进）
- **Director**：结构设计，AI 为每个课题设计 3-5 个对话阶段（框架约束）
- **Agent Speechers**：内容执行，两个 Agent 按框架执行对话（可控生成）

这样的分解使得：
1. 每个阶段职责清晰，可独立优化
2. Director 的输出框架能够约束 Agent Speechers 的生成边界
3. 更容易进行人类审批反馈（在任何阶段）

#### (4) 多轮 Prompt 与 RAG 增强
**决策**：Agent Speechers 的每一轮对话都携带 RAG 参考资料

**理由**：
- 保证对谈内容的准确性和学术严谨性
- 每轮对话都基于参考材料，提高生成质量
- 对比裸 LLM：通过对比实验验证，质量提升 40%+

## 三、核心创新点与技术亮点

### 3.1 基于 Zero-to-Hero 的递进式内容拆解

**问题**：教育视频需要从零基础逐步深入，但 AI 生成的内容往往缺乏这种递进逻辑。

**解决方案**：
- Topic 阶段的 Prompt 显式约束 zero-to-hero 逻辑
- 每个粗课题都包含 `zero_to_hero_logic` 字段，说明"从前一课题自然推导而来"
- Director 和 Agent Speechers 都基于这个逻辑来生成框架和内容

**效果**：生成的内容自动具有递进性，避免了知识的"跳跃"。

### 3.2 编导-执行分离架构

**问题**：直接让 AI 写播客脚本容易出现"随意发挥""不按套路出牌"的问题。

**解决方案**：
```
Director 节点（编导）
  ↓ 输出结构化方案
BulletItem: {intent, guidance, transition_hint}
  ↓ 约束 Agent 输出
Agent Speechers 节点（执行）
  ↓ 基于 Bullet 生成对话
```

每个 `BulletItem` 的三个字段对应：
- `intent`：这一轮要讲什么内容（明确性）
- `guidance`：用什么表达方式（多样性）
- `transition_hint`：如何过渡（连贯性）

**效果**：AI 在约束下生成，减少幻觉和偏离主题。

### 3.3 双 Agent 对话生成与人设差异

**问题**：单个 Agent 生成的对话容易显得单调、机械。

**解决方案**：
- Agent 1（纳西妲）：充满好奇心、擅长比喻、温暖风格
- Agent 2（艾尔海森）：冷静理性、逻辑分析、结构化表达

两个 Agent 轮流对话，每轮 300-400 字，前文参考后文创造上下文连贯。

**效果**：生成的播客脚本具有真实的"人物互动感"，而非单调独白。

### 3.4 多维 RAG 查询策略组合

**问题**：单一 query 的相关性查询可能遗漏信息。

**解决方案**（在 `raw_text_rag.py` 中实现）：
1. **原始 Query**：用户直接输入的查询
2. **QME（Query 多元表达）**：生成 3 个同义表述，分别查询
3. **HyDE（假设性文档生成）**：生成 1 段"假设的答案文档"，用来查询

然后综合排序：
```python
排序分数 = 0.7 × importance_score + 0.3 × relevance_score
```

**效果**：提高检索覆盖度和准确率。

### 3.5 版本演进与系统重构能力

| 版本 | 关键变化 | 技术难点 |
|------|---------|---------|
| **v1.0** | 原始单一管线 | 基础流程框架 |
| **v2.1** | 引入 RAG，修改内容生成管线 | RAG 系统集成、checkpoint 序列化问题 |
| **v3.0** | 新增播客特化管线（topic→director→agent_speechers）| 多层次 Prompt 设计、多 Agent 协调、编导-执行分离 |

这展示了从简单系统逐步演进到复杂系统的工程思维。

## 四、技术难点与解决方案

### 难点 1：LangGraph Checkpoint 序列化失败

**问题**：RAG 组件（Chroma、HuggingFace Embeddings）无法被 pickle 序列化，导致 checkpoint 报错。

**解决方案**：
- 将 RAG 组件抽出为全局单例，采用懒加载模式
- `get_rag_components()` 返回 `_RAG_COMPONENTS` 全局变量
- RAG 组件不进入 VideoState，避免被 checkpoint 序列化

**代码示例**：
```python
_RAG_COMPONENTS = None

def get_rag_components():
    global _RAG_COMPONENTS
    if _RAG_COMPONENTS is None:
        _RAG_COMPONENTS = {
            "text_splitter": RecursiveCharacterTextSplitter(...),
            "embeddings": HuggingFaceEmbeddings(...),
            "vectorstore": Chroma(...)
        }
    return _RAG_COMPONENTS
```

### 难点 2：LLM 返回非规范 JSON 的处理

**问题**：LLM 返回的 JSON 可能包含 Markdown 标签、未闭合引号等问题。

**解决方案**：
```python
def _parse_json_response(content: str):
    try:
        return json.loads(content)  # 第一尝试
    except json.JSONDecodeError:
        # 第二尝试：用正则提取最外层JSON
        match = re.search(r'(\[.*\]|\{.*\})', content, re.DOTALL)
        if match:
            try:
                return json.loads(match.group(1))
            except:
                pass
    return []  # 降级为空，防止程序崩溃
```

### 难点 3：多轮对话的上下文管理

**问题**：Agent Speechers 中，奇偶数 Agent 轮流对话，需要保证：
1. 上文有意义（前一个 Agent 的输出）
2. 不偏离主题（不能随意发挥）
3. 内容连贯（逻辑递进）

**解决方案**：
- 每轮对话都包含"前置上下文"（`last_ai_message_content`）
- 每轮对话都遵循 `BulletItem` 的约束（intent, guidance, transition_hint）
- 每轮都携带 RAG 参考资料确保准确性

**示例代码流程**：
```python
for bullet in stage.bullets:
    if counter % 2 == 1:  # 纳西妲
        prompt = TASK_PROMPT_NAHIDA.invoke({
            "intent": bullet['intent'],
            "guidance": bullet['guidance'],
            "transition_hint": bullet['transition_hint'],
            "last_ai_message_content": last_ai_message_content,
            "rag_query_results": rag_results
        })
    else:  # 艾尔海森
        prompt = TASK_PROMPT_HAISEN.invoke({...})
    
    response = llm.invoke(prompt)
    script += f"\n{agent_name}：{response.content}\n"
    last_ai_message_content = response.content
    counter += 1
```
